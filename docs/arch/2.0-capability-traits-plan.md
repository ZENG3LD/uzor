# uzor 2.0 — Capability-traits Refactor Plan

**Date**: 2026-05-16
**Branch**: `paint-primitives-2026-05`
**Status**: planning — Phase 0 charter pending
**Drivers**: 6 primitive-extension asks from `nemo/uzor-tessera-paint/docs/handoff/`

---

## Philosophy

`RenderContext` накапливал capabilities ~год до состояния "fat trait" (50+ методов, ~810 LOC).
Каждая новая ручка увеличивает порог для нового backend, размывает API surface, путает
stateful vs stateless семантику. **6 asks от paint team — это симптом, не диагноз.**
Диагноз: trait делает слишком много одновременно.

**Целевая архитектура:** `RenderContext` становится supertrait-композицией ~10 узких
capability traits. Stateless queries отделяются физически. Opt-in capabilities (blur,
image) backends объявляют явно, а не симулируют no-op'ами.

**uzor 2.0 — major bump.** Ломаем downstream consumers (uzor-tessera, tessera-paint,
uzor-examples, mlc-landing) под нашим контролем — миграция писательна не больше чем
landing самого carve'а. Pain is OK if architecture wins.

---

## Audit — текущая боль

| Грязь | Проявление |
|---|---|
| Fat trait | 50+ методов в одном `RenderContext`, новые backends обязаны импл'ить всё |
| Silent capability gaps | `draw_blur_background` — no-op default на tiny-skia (молча); консумер не знает что blur отсутствует |
| Stateful/stateless смешение | `measure_text(&self, text)` stateful, новый `text_bounds(&self, text, font)` stateless — асимметрия baked-in |
| Дубликаты API | Spec просит `push_clip_path` хотя `push_mask`/`pop_mask` уже есть |
| Связь "compile-time expand" с full ctx | paint `expand()` берёт `&dyn RenderContext` чтобы прочитать text bounds — overkill |

## Целевая таксономия (Phase 2 carve)

```rust
// Core — must-have для любого backend
trait Painter         // begin_path/move_to/line_to/stroke/fill/transforms
trait TextRenderer    // fill_text/stroke_text/set_font (stateful)
trait Masking         // push_mask/pop_mask/clip + push_clip_svg_path (NEW)
trait Effects         // set_shadow/clear_shadow/set_blend_mode

// Helpers — default impls поверх Painter
trait ShapeHelpers    // fill_rect/fill_rounded_rect/stroke_rounded_rect_corners
trait BatchPainter    // draw_line_batch/draw_circle_batch/stroke_polyline (NEW)
trait GradientPainter // fill_linear_gradient/fill_radial_gradient

// Stateless queries — отдельный trait, paint expand() берёт ИМЕННО его
trait TextMetrics     // measure_text/text_bounds (NEW)/measure_text_glyphs (NEW)

// Opt-in capabilities — backends объявляют явно
trait BackdropBlur    // draw_blur_background (no default impl)
trait ImagePainter    // draw_image_rgba (no default impl)

// Compound — то что было RenderContext
trait RenderContext:
    Painter + TextRenderer + TextMetrics + Masking + Effects +
    ShapeHelpers + BatchPainter + GradientPainter
{
    fn dpr(&self) -> f64;
}
```

Consumer-side: `&dyn RenderContext` работает как раньше (supertrait). Кто хочет только
metrics — берёт `&dyn TextMetrics`. Backends объявляют capability явно
(`impl BackdropBlur for TinySkia { ... }` либо не объявляют — компилятор гарантирует
что consumer знает разницу).

## Фазы

### Phase 0 — Audit & charter (1д)
- Перебрать все 50+ методов `RenderContext` → bucket по traits
- Решить opt-in/required boundaries (`BackdropBlur`, `ImagePainter` — opt-in)
- Договориться об именах (`Painter` vs `Canvas` vs `Surface`)
- Доковать миграционные шаги для каждого downstream consumer
- Артефакт: `uzor/docs/arch/2.0-capability-traits-charter.md`

### Phase 1 — Бэкенд-баги (parallel, zero API risk)
Можно слить параллельно Phase 0 — не зависят от carve'а.
- **#2** vello-gpu shadow fix (`ShapeIntent::{Fill, Stroke{width}}` в `emit_shadow_for_shape`)
- **#3** tiny-skia native blur (3-pass box-blur, scratch `Pixmap`, ~50 LOC)

Коммитятся в `paint-primitives-2026-05`, без bump'а в crates.io (внутренние исправления).

### Phase 2 — Capability carve (THE break)
- Carve 10 traits согласно таксономии
- Перенос default impls на правильные traits
- Re-impl всех 3 активных backends под новую таксономию
- Sweep call sites в `uzor/uzor/src/ui/effects/`, `cursor/`, `panels/`, `numbers/`
- Gate: `cargo check --workspace` + `cargo test --workspace` зелёные
- **Без bump'а в crates.io** — branch ещё не merged

### Phase 3 — TextMetrics ship (#6 + surface для #1)
- `text_bounds(text, font) → TextBounds` — реальный impl на cosmic-text
- `measure_text_glyphs(text, font) → Vec<GlyphMetric>` — stub impl (per-char), declaration реальная
- Paint unblocks: `emit_per_glyph_layers` против trait surface

### Phase 4 — Glyph shaper backbone (#1 real)
- Cosmic-text wiring (или parley если уже transitive)
- Backend cache `Mutex<HashMap<(font_hash, text), Vec<GlyphMetric>>>`
- Cluster correctness: graphemes, emoji ZWJ, CJK
- Acceptance: 5/5 acceptance criteria из spec'а

### Phase 5 — Clip-to-path (#5)
- `Masking::push_clip_svg_path(d: &str)` с default impl через existing `push_mask`
- Free function `uzor::core::path::emit_svg_path(ctx, d)` — публично reusable
- vello-gpu native override (`scene.push_clip(BezPath)`)
- `pop_clip` **не существует** — `pop_mask` единственный путь

### Phase 6 — Batch primitives (#4)
- `BatchPainter::{draw_line_batch, draw_circle_batch, stroke_polyline}`
- `LineSegment`, `CircleBatch` в `uzor::core::render::types`
- Default impls (loops)
- Backend overrides: tiny-skia merged Path/single stroke, vello-gpu один encoder fragment
- Performance gate: tiny-skia 200-line batch ≥5× быстрее N individual calls

### Phase 7 — Migration sweep + smoke
- uzor-tessera: миграция сигнатур
- tessera-paint: path-dep на branch с Phase 2, дальнейшие фазы подхватываются
- uzor-examples: все demo binaries компилируются
- mlc-landing: scan на `RenderContext` usage, fix call sites
- Smoke: запуск tessera-paint-debug на каждой фазе, screenshot + trace

### Phase 8 — 2.0 release
- Bump workspace `1.2.2 → 2.0.0`
- `cargo publish` по dependency order (uzor-fonts → uzor → backends → window/desktop)
- CHANGELOG с migration guide
- Tag `v2.0.0`

## Что НЕ делаю в 2.0

- **Не переезжаю на builder-style Vello Scene API** — это уже не uzor, другая абстракция. Если когда-то — uzor 3.0.
- **Не убиваю stateful `fill_text` / `measure_text`** — Canvas2D-style императивный API остаётся осознанным выбором для embed-friendly trait. Stateless только для queries.
- **Не добавляю фичи вне 6 asks**: path booleans, OKLab gradients, glyph outline extraction, stroke patterns — следующая major.
- **Не разбиваю Painter дальше** на PathBuilder + Filler + Stroker. 9-10 traits — комфортный потолок; больше = карго-культ.

## Риски и митигация

| Риск | Митигация |
|---|---|
| 2.0 break ломает crates.io users вне нашего контроля | 1.2.2 опубликован — pin'у'тся consumers остаются на 1.x. `release-1.x` branch для критичных fix'ов 3 месяца. После — EOL. |
| Phase 2 carve пропускает edge case | Smoke на каждом backend после carve'а: tessera-paint-debug на tiny-skia + vello-gpu + vello-cpu, screenshot diff с baseline'ом из 1.2.2 |
| Phase 4 cosmic-text не лезет на каком-то backend | Fallback: per-char approximation, gate Phase 4 как "best-effort". Принято в acceptance criteria. |
| Downstream migration больше работы чем кажется | Phase 7 — отдельная фаза с buffer'ом. tessera-paint flips path-dep на branch с Phase 2 — раннее testing. |
| `BackdropBlur` opt-in ломает consumers которые ожидали no-op | По дизайну: тихий no-op хуже чем compile error. Consumers явно решают fallback. |
