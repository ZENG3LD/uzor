## Open Questions (resolve before Phase 2 starts)

1. **`draw_hover_rect` / `draw_active_rect` / `draw_sidebar_*` / `draw_glass_button_3d`** — these
   are UI-pattern helpers that depend on blur state. They belong in `ShapeHelpers` by structure but
   call `draw_blur_background` (opt-in `BackdropBlur`). Option A: move them to a separate
   `UiEffectHelpers` trait with default impls that degrade gracefully. Option B: keep in
   `ShapeHelpers` with default impls that branch on `has_blur_background()` — but that ties
   `ShapeHelpers` to `BackdropBlur` at the trait bound level. **Decision needed from owner.**

2. **`RenderContextExt` fate** — the `set_blur_image` / `set_use_convex_glass_buttons` helpers on
   `RenderContextExt` are backend setup calls (not drawing). They do not fit into any of the 10
   capability traits. Options: keep as-is (separate setup trait); absorb into `BackdropBlur` as
   optional associated setup methods; delete and make backends expose typed setters directly.

3. **`draw_image` (URL-based) vs `draw_image_rgba` (raw bytes)** — current `draw_image` (url,
   x,y,w,h) returns `bool` and has a `false` default. `draw_image_rgba` is the actually
   implemented one. Should `ImagePainter` cover both, or promote only `draw_image_rgba` to
   opt-in and leave `draw_image` as a default no-op stub in `RenderContext` proper?
   Recommendation: `ImagePainter` gets both; `draw_image` stays with `-> bool` signature;
   backends that don't cache URLs just return `false`.

4. **`BlendMode` enum ownership** — currently defined inline in `context.rs`. In 2.0 it belongs in
   `uzor::core::render::types`. Confirm this is the chosen location.

5. **`stroke_text`** — `VelloGpuRenderContext` has an empty `stroke_text` body (text stroking not
   implemented). Should `TextRenderer` require it as abstract (forcing backends to provide a stub)
   or provide a `fn stroke_text(&mut self, _: &str, _: f64, _: f64) {}` default?
   Recommendation: default no-op (same pattern as current) — `fill_text` is the real primitive.

```rust
/// Core drawing trait — path construction, fill/stroke, transforms, state.
///
/// Every backend must implement this. No opt-out methods.
pub trait Painter {
    // State
    fn save(&mut self);
    fn restore(&mut self);

    // Transform
    fn translate(&mut self, x: f64, y: f64);
    fn rotate(&mut self, angle: f64);
    fn scale(&mut self, x: f64, y: f64);

    // Style setters
    fn set_fill_color(&mut self, color: &str);
    fn set_global_alpha(&mut self, alpha: f64);
    fn set_stroke_color(&mut self, color: &str);
    fn set_stroke_width(&mut self, width: f64);
    fn set_line_dash(&mut self, pattern: &[f64]);
    fn set_line_cap(&mut self, cap: &str);
    fn set_line_join(&mut self, join: &str);

    // Style helpers (default impls)
    fn set_fill_color_alpha(&mut self, color: &str, alpha: f64) {
        self.set_fill_color(color);
        self.set_global_alpha(alpha.clamp(0.0, 1.0));
    }
    fn reset_alpha(&mut self) { self.set_global_alpha(1.0); }

    // Path construction
    fn begin_path(&mut self);
    fn move_to(&mut self, x: f64, y: f64);
    fn line_to(&mut self, x: f64, y: f64);
    fn close_path(&mut self);
    fn rect(&mut self, x: f64, y: f64, w: f64, h: f64);
    fn arc(&mut self, cx: f64, cy: f64, radius: f64, start_angle: f64, end_angle: f64);
    fn ellipse(
        &mut self,
        cx: f64, cy: f64, rx: f64, ry: f64,
        rotation: f64, start: f64, end: f64,
    );
    fn quadratic_curve_to(&mut self, cpx: f64, cpy: f64, x: f64, y: f64);
    fn bezier_curve_to(
        &mut self,
        cp1x: f64, cp1y: f64,
        cp2x: f64, cp2y: f64,
        x: f64, y: f64,
    );

    // Commit
    fn stroke(&mut self);
    fn fill(&mut self);
}

/// Text rendering — stateful (current font applies).
///
/// Separated from `Painter` so consumers that only draw shapes
/// do not need to implement glyph layout.
pub trait TextRenderer {
    fn set_font(&mut self, font: &str);
    fn set_text_align(&mut self, align: TextAlign);
    fn set_text_baseline(&mut self, baseline: TextBaseline);
    fn fill_text(&mut self, text: &str, x: f64, y: f64);

    /// Stroke text outline. Backends without native stroke-text may provide a no-op.
    fn stroke_text(&mut self, text: &str, x: f64, y: f64) {
        let _ = (text, x, y);
    }

    /// Fill text with rotation around anchor. Default uses save/translate/rotate/fill_text/restore.
    /// Backends (vello-gpu) override for pre-rotation baseline correctness.
    fn fill_text_rotated(&mut self, text: &str, x: f64, y: f64, angle: f64)
    where
        Self: Painter,
    {
        if angle.abs() < 0.001 {
            self.fill_text(text, x, y);
        } else {
            self.save();
            self.translate(x, y);
            self.rotate(angle);
            self.fill_text(text, 0.0, 0.0);
            self.restore();
        }
    }

    /// Fill text centered at position.
    fn fill_text_centered(&mut self, text: &str, x: f64, y: f64) {
        self.set_text_align(TextAlign::Center);
        self.set_text_baseline(TextBaseline::Middle);
        self.fill_text(text, x, y);
    }
}

/// Stateless text measurement — can be taken as `&dyn TextMetrics` without a mutable context.
///
/// Paint `expand()` calls and layout passes take this trait, not the full `RenderContext`.
/// Backends implement this alongside `TextRenderer`.
pub trait TextMetrics {
    /// Width of `text` in current font (pixels). Uses current font state.
    fn measure_text(&self, text: &str) -> f64;

    // Phase 3 additions (declare here, implement in Phase 3):
    // fn text_bounds(&self, text: &str) -> TextBounds;
    // fn measure_text_glyphs(&self, text: &str) -> Vec<GlyphMetric>;
}

/// Clipping and mask layers.
pub trait Masking {
    /// Clip subsequent draws to the current path (binary clip).
    fn clip(&mut self);

    /// Convenience: clip to a rectangle.
    fn clip_rect(&mut self, x: f64, y: f64, width: f64, height: f64)
    where
        Self: Painter,
    {
        self.begin_path();
        self.rect(x, y, width, height);
        self.clip();
    }

    /// Push a mask layer using the current path.
    /// Default: save + clip (binary). Vello may override with alpha-mask layer.
    fn push_mask(&mut self)
    where
        Self: Painter,
    {
        self.save();
        self.clip();
    }

    /// Pop the most recently pushed mask layer.
    fn pop_mask(&mut self)
    where
        Self: Painter,
    {
        self.restore();
    }
}

/// Rendering effects — shadow, blend mode.
///
/// Backends without native support provide no-op defaults.
/// These are NOT opt-in traits because they have functioning
/// default no-ops (unlike BackdropBlur which silently corrupts UX).
pub trait Effects {
    fn set_shadow(&mut self, dx: f64, dy: f64, blur: f64, color: &str) {
        let _ = (dx, dy, blur, color);
    }
    fn clear_shadow(&mut self) {}
    fn set_blend_mode(&mut self, mode: BlendMode) {
        let _ = mode;
    }
}

/// Shape convenience helpers — all have default impls over `Painter`.
///
/// Backends may override (e.g. vello-gpu overrides `rounded_rect_corners`
/// with native kurbo rounded-rect for better precision).
pub trait ShapeHelpers: Painter {
    fn stroke_rect(&mut self, x: f64, y: f64, w: f64, h: f64);
    fn fill_rect(&mut self, x: f64, y: f64, w: f64, h: f64);

    fn rounded_rect(&mut self, x: f64, y: f64, w: f64, h: f64, r: f64) {
        // default: arc-based path (existing impl)
    }

    fn rounded_rect_corners(
        &mut self,
        x: f64, y: f64, w: f64, h: f64,
        tl: f64, tr: f64, br: f64, bl: f64,
    ) {
        let r = [tl, tr, br, bl].iter().copied().fold(f64::INFINITY, f64::min);
        let r = r.max(0.0).min((w / 2.0).min(h / 2.0));
        self.rounded_rect(x, y, w, h, r);
    }

    fn fill_rounded_rect(&mut self, x: f64, y: f64, w: f64, h: f64, radius: f64) {
        self.begin_path();
        self.rounded_rect(x, y, w, h, radius);
        self.fill();
    }

    fn stroke_rounded_rect(&mut self, x: f64, y: f64, w: f64, h: f64, radius: f64) {
        self.begin_path();
        self.rounded_rect(x, y, w, h, radius);
        self.stroke();
    }

    fn fill_rounded_rect_corners(
        &mut self,
        x: f64, y: f64, w: f64, h: f64,
        tl: f64, tr: f64, br: f64, bl: f64,
    ) {
        self.begin_path();
        self.rounded_rect_corners(x, y, w, h, tl, tr, br, bl);
        self.fill();
    }

    fn stroke_rounded_rect_corners(
        &mut self,
        x: f64, y: f64, w: f64, h: f64,
        tl: f64, tr: f64, br: f64, bl: f64,
    ) {
        self.begin_path();
        self.rounded_rect_corners(x, y, w, h, tl, tr, br, bl);
        self.stroke();
    }
}

/// Gradient fills — default impls fall back to first-stop solid fill.
pub trait GradientPainter: Painter {
    fn fill_linear_gradient(
        &mut self,
        stops: &[(f32, &str)],
        x1: f64, y1: f64,
        x2: f64, y2: f64,
    ) {
        let _ = (x1, y1, x2, y2);
        if let Some((_, color)) = stops.first() {
            self.set_fill_color(color);
            self.fill();
        }
    }

    fn fill_radial_gradient(
        &mut self,
        cx: f64, cy: f64, r: f64,
        stops: &[(f32, &str)],
        x: f64, y: f64, w: f64, h: f64,
    ) {
        let _ = (cx, cy, r);
        if let Some((_, color)) = stops.first() {
            self.set_fill_color(color);
            // ShapeHelpers::fill_rect not accessible here without supertrait bound;
            // in Phase 2 carve, add ShapeHelpers as supertrait or inline the begin_path+rect+fill.
            self.begin_path();
            self.rect(x, y, w, h);
            self.fill();
        }
    }
}

/// Backdrop blur — GPU-only opt-in capability.
///
/// Backends that do not implement this trait are statically known
/// to callers — no silent no-op. Tessera widget paint functions that
/// want blur check `is<dyn BackdropBlur>` or use capability downcast.
///
/// VelloGpuRenderContext implements this. TinySkia does NOT (until Phase 1 #3).
pub trait BackdropBlur {
    /// Draw the blurred background texture clipped to the given rect.
    fn draw_blur_background(&mut self, x: f64, y: f64, width: f64, height: f64);

    /// Returns true when a blur image is loaded and ready.
    fn has_blur_background(&self) -> bool;

    /// Returns true when blur is active AND convex glass button style is enabled.
    fn use_convex_glass_buttons(&self) -> bool;

    /// Draw a 3D convex glass button. Default: blur background + fill_rounded_rect.
    /// VelloGpu overrides with full specular/gradient/shadow 3D effect.
    fn draw_glass_button_3d(
        &mut self,
        x: f64, y: f64, width: f64, height: f64,
        radius: f64, is_active: bool, color: &str,
    );
}

/// Image rendering — opt-in.
///
/// Backends that cannot render images do not implement this trait.
/// Callers check capability explicitly instead of silently skipping.
pub trait ImagePainter {
    /// Draw a cached image by URL/id. Returns false if not yet loaded.
    fn draw_image(
        &mut self,
        image_id: &str,
        x: f64, y: f64,
        width: f64, height: f64,
    ) -> bool;

    /// Draw raw RGBA pixel data.
    fn draw_image_rgba(
        &mut self,
        data: &[u8],
        img_width: u32,
        img_height: u32,
        x: f64, y: f64,
        width: f64, height: f64,
    );
}

/// UI-effect convenience helpers — composite methods that branch on BackdropBlur availability.
///
/// NEW trait, not in the original plan taxonomy. Resolves Open Question #1.
/// All methods have default impls. Backends that implement BackdropBlur get glass effects
/// automatically; others get solid fallbacks.
///
/// Tessera widgets call these instead of branching on `has_blur_background()` themselves.
pub trait UiEffectHelpers: ShapeHelpers {
    fn draw_hover_rect(&mut self, x: f64, y: f64, width: f64, height: f64, color: &str) {
        self.set_fill_color(color);
        self.fill_rect(x, y, width, height);
    }

    fn draw_active_rect(&mut self, x: f64, y: f64, width: f64, height: f64, color: &str) {
        self.set_fill_color(color);
        self.fill_rect(x, y, width, height);
    }

    fn draw_hover_rounded_rect(
        &mut self,
        x: f64, y: f64, width: f64, height: f64,
        radius: f64, color: &str,
    ) {
        self.set_fill_color(color);
        self.fill_rounded_rect(x, y, width, height, radius);
    }

    fn draw_active_rounded_rect(
        &mut self,
        x: f64, y: f64, width: f64, height: f64,
        radius: f64, color: &str,
    ) {
        self.set_fill_color(color);
        self.fill_rounded_rect(x, y, width, height, radius);
    }

    fn draw_sidebar_hover_item(
        &mut self,
        x: f64, y: f64, width: f64, height: f64,
        accent_color: &str, bg_color: &str,
        indicator_width: f64,
    ) {
        self.set_fill_color(accent_color);
        self.fill_rect(x, y, indicator_width, height);
        self.draw_hover_rect(x + indicator_width, y, width - indicator_width, height, bg_color);
    }

    fn draw_sidebar_active_item(
        &mut self,
        x: f64, y: f64, width: f64, height: f64,
        accent_color: &str, bg_color: &str,
        indicator_width: f64,
    ) {
        self.set_fill_color(accent_color);
        self.fill_rect(x, y, indicator_width, height);
        self.draw_active_rect(x + indicator_width, y, width - indicator_width, height, bg_color);
    }
}

/// Compound trait — the full rendering surface.
///
/// `&dyn RenderContext` works identically to today's API.
/// Backends that implement BackdropBlur or ImagePainter declare that separately.
pub trait RenderContext:
    Painter
    + TextRenderer
    + TextMetrics
    + Masking
    + Effects
    + ShapeHelpers
    + GradientPainter
    + UiEffectHelpers
{
    /// Device pixel ratio for crisp rendering.
    fn dpr(&self) -> f64;
}
```

## 4. Opt-in Capabilities Analysis

### Confirmed opt-in: `BackdropBlur` + `ImagePainter`

Per the plan. Rationale: both have silent no-op defaults today that cause invisible UX gaps
(blur region renders nothing, image region renders nothing). Converting to explicit opt-in traits
gives consumers a compile-time guarantee or a deliberate downcast.

### Additional opt-in candidates from the silent no-op survey

| Method | Current default | Backend that implements | Is it a silent gap? | Recommendation |
|--------|----------------|------------------------|---------------------|----------------|
| `set_shadow` / `clear_shadow` | no-op | VelloGpu partially (Phase 1 #2 fixes it) | Yes — shadow silently absent on tiny-skia | Keep in `Effects` with default no-op. **Not opt-in.** Rationale: shadow is decorative, its absence does not corrupt data. Contrast with blur where absence changes readability of overlaid content. |
| `set_blend_mode` | no-op | VelloGpu (M6-P3) | Yes — blending silently Normal on other backends | Same reasoning as shadow — keep in `Effects` with no-op default. Consumer can query capability if needed in future. |
| `draw_image` | returns false | None (stub only) | No — caller checks return value | In `ImagePainter` opt-in. |
| `draw_image_rgba` | no-op | VelloGpu, canvas2d | Yes — pixel data silently discarded | In `ImagePainter` opt-in. |
| `stroke_text` | abstract (required) but VelloGpu body is empty | None | Mild gap | Default no-op in `TextRenderer`. Not opt-in — rare use case. |

**Conclusion**: only `BackdropBlur` and `ImagePainter` become opt-in. `Effects` keeps default
no-ops because their absence is visually tolerable and the no-op behaviour is well-understood.

### `UiEffectHelpers` — new opt-in-adjacent design

The 6 UI hover/active/glass methods currently live in `RenderContext` and call
`has_blur_background()` at runtime. In 2.0 the plan is:

- Move them to `UiEffectHelpers` supertrait of `RenderContext`.
- Default impls: solid colour only (no blur).
- Backends that implement `BackdropBlur` **additionally** override `draw_hover_rect` etc. to
  inject `draw_blur_background` calls. This is done in a blanket `impl<T: BackdropBlur + ShapeHelpers> UiEffectHelpers for T { ... }` — giving backends the glass variants for free when they
  declare `BackdropBlur`.
- This eliminates the `has_blur_background()` runtime branch in the default impls entirely.

## 7. Summary of Decisions

1. **11 traits** total (plan said ~10; `UiEffectHelpers` is the addition).
2. `BatchPainter` deferred to Phase 6 — not in `RenderContext` supertrait until then.
3. `Effects` keeps default no-ops — not opt-in. Shadow/blend absence is tolerable.
4. `UiEffectHelpers` replaces the 6 `draw_hover_*` / `draw_sidebar_*` / `draw_glass_*` methods.
   Blanket impl for `BackdropBlur + ShapeHelpers` backends provides glass variants automatically.
5. `BlendMode` enum moves to `effects.rs`.
6. `RenderContextExt` stays as-is pending Open Question #2 resolution.
7. `stroke_text` gets a default no-op in `TextRenderer`.
8. `draw_image` stays in `ImagePainter` with `-> bool` signature.
9. `fill_text_rotated` / `fill_text_centered` default impls require `where Self: Painter` bound
   on the `TextRenderer` methods — standard Rust AFIT-era pattern.
10. `measure_text` stays as stateful query in `TextMetrics` (uses current font state). Phase 3
    adds true stateless `text_bounds(text, font)`.
